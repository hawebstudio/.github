# HA Web Studio Organization Recommendations

## Repository Structure

| Repository | Purpose |
| --- | --- |
| `website` | Main HA Web Studio website |
| `ui` | Shared UI components, tokens, and patterns |
| `templates` | Reusable website and landing page templates |
| `docs` | Public documentation and knowledge base |
| `brand-assets` | Logos, banners, social media images, and brand source files |
| `portfolio` | Case study and portfolio source |
| `tools` | Internal audit scripts, automation, and utilities |
| `boilerplates` | Opinionated foundations for repeated project types |
| `starter-nextjs` | Next.js starter project |
| `starter-wordpress` | WordPress starter structure |
| `starter-shopify` | Shopify starter structure |

## Branding Recommendations

- Keep the GitHub organization avatar simple, legible, and recognizable at small sizes.
- Use one banner system across GitHub, LinkedIn, YouTube, and website hero graphics.
- Maintain light and dark variants of the logo and banner.
- Add a verified contact email only after the mailbox is active.
- Keep README colors close to GitHub-native neutrals with a restrained blue accent.

## GitHub Profile Improvements

- Pin the six most important repositories on the organization profile: `website`, `ui`, `templates`, `docs`, `tools`, and `portfolio`.
- Add repository descriptions, topics, licenses, and short README files before making new repositories public.
- Add issue templates, pull request templates, security policy, contributing guide, and code of conduct to public repositories.
- Add GitHub Actions for Markdown linting, link checks, and dependency checks.
- Add live GitHub Readme Stats cards only after public repositories exist and the cards return useful data.
